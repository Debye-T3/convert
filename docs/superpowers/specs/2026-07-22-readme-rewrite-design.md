# README 重写设计

## 目标

将仓库根目录的 `README.md` 重写为一份中文实验室内部使用手册，同时兼顾项目维护者。文档应让首次接触该工具的实验人员能够完成 ARPES/Scienta DA30 数据转换，并让开发者能够找到环境、测试、构建和代码结构信息。

## 目标读者与语言

- 主要读者：使用转换工具的实验人员。
- 次要读者：维护、测试和打包该项目的开发者。
- 正文使用中文；界面按钮、参数键、命令、文件扩展名和技术名称保留英文。
- 语气采用实验室操作手册风格，强调具体步骤、输入、输出和注意事项。

## 信息结构

README 按以下顺序组织：

1. 项目名称与一句话定位。
2. 核心能力与支持格式速览。
3. 环境要求和首次启动。
4. 四步转换工作流：
   - 选择 `.txt`、`.pxt`、`.pxp` 或 `.zip` 文件；
   - 设置批次默认值、逐文件覆盖值和自定义字段；
   - 选择输出目录并批量转换；
   - 查看生成的 H5 文件结构。
5. Excel 参数模板导出与导入。
6. 输入格式和特殊读取规则。
7. 输出文件、命名规则及 xarray/ERLab 兼容性。
8. 常见问题、当前限制与故障排查。
9. 面向维护者的开发环境、测试、打包和项目结构。

用户操作内容放在前半部分，开发资料放在后半部分。避免重复叙述同一命令或规则。

## 截图方案

- 图片存放在 `docs/images/readme/`，采用 PNG 格式。
- 使用统一窗口尺寸，分别截取当前 GUI 的四个标签页：
  1. `Select Files`
  2. `Parameters`
  3. `Convert`
  4. `H5 Info`
- 截图不得包含个人绝对路径、真实样品名称或其他实验敏感信息。
- 如果仓库中没有适合公开展示的输入样例，则使用干净的空白界面，不伪造转换结果。
- README 中在相应操作步骤附近放置截图，并提供简洁、可读的替代文本。

## 内容准确性约束

README 只描述当前代码中已经可用的行为，并明确以下细节：

- GUI 当前包含四个标签页：文件选择、参数设置、转换和 H5 信息。
- 支持 `.txt`、`.pxt`、`.pxp` 和 DA30 `.zip` 输入。
- 当 `.pxt` 旁存在同名 `.txt` 时，转换引擎优先读取该 `.txt`；否则使用 DA30 PXT 读取器。
- PXT 读取支持通道选择、自动通道、暗通道扣除及能量轴/角度轴覆盖参数。
- 参数页支持批次默认值、逐文件覆盖、自定义字段以及 Excel 模板导出/导入。
- 输出文件重名时自动追加 `_1`、`_2` 等编号，不覆盖已有文件。
- 单区域数据通常写为 `xarray.DataArray` 或 `xarray.Dataset`；多区域 `.pxp`/`.zip` 数据可能写为 `xarray.DataTree`。
- 后端包含 PNG 预览能力，但当前主窗口未接入 `PreviewTab`，转换线程也显式禁用预览；因此不得将预览描述为现有 GUI 功能。
- `run.bat` 和 `build_exe.bat` 包含本机绝对路径。README 应优先提供可移植的 Conda 命令，并将批处理脚本的路径限制写入注意事项。
- PyInstaller 生成 onedir 程序；分发时必须复制整个 `dist\converter_app` 目录，而不是只复制可执行文件。

## 命令与示例

README 至少提供并核对以下命令：

```bat
conda env create -f environment.yml
conda env update -n convert-da30 -f environment.yml
conda run -n convert-da30 python converter_app.py
conda run -n convert-da30 python -m unittest discover -s tests
build_exe.bat
```

示例不得暗示仓库包含实际不存在的发布包、安装程序、示例数据或完整测试套件。

## 验证标准

完成重写后执行以下检查：

1. 对照 `converter/` 和 `gui/` 核查支持格式、工作流、参数行为和输出规则。
2. 验证 README 中引用的所有本地文件与图片路径存在。
3. 运行现有 `unittest` 测试。
4. 检查 Markdown 标题层级、代码块、图片链接和表格的可读性。
5. 检查文档中没有本机隐私路径、虚构功能、占位文本或未完成事项。

## 非目标

- 不修改转换逻辑、GUI 行为、批处理脚本或依赖配置。
- 不新增预览入口、示例实验数据、安装程序或 GitHub Release。
- 不把构建产物和转换数据加入版本控制。
